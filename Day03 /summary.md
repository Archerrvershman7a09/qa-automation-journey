# Day 03 Summary

## What I learned

Today I learned how to write basic test cases and bug reports.

A test case describes what should be tested, which steps should be followed, and what result is expected.

A bug report describes an issue found during testing and helps developers reproduce and fix the problem.

## Key Concepts

### Test Case
A test case includes:
- Title
- Prerequisites
- Steps
- Expected Result
- Actual Result
- Status

### Bug Report
A bug report includes:
- Title
- Severity
- Steps to Reproduce
- Expected Result
- Actual Result
- Attachments

### Positive Test
A positive test checks that the system works correctly with valid data.

Example:
User logs in with valid email and valid password.

### Negative Test
A negative test checks how the system behaves with invalid data.

Example:
User tries to log in with an invalid password.

## Practical Task

I created:
- 5 test cases for the login feature
- 2 sample bug reports for login-related issues

## What was difficult

The most difficult part was understanding the difference between Expected Result and Actual Result.

Now I understand it this way:

Expected Result = what should happen.

Actual Result = what really happened during testing.
